## 参加の前に

このプロジェクトは以下の方針で運営されます：

- **最終決定権は著者（馬場祐平）にあります** — 全 PR の最終マージ可否は著者が判断します
- **共同創作者（K氏/しおめ）もレビュー権限を持ちます**
- **商業出版を視野に入れています** — そのため、マージされたコンテンツの権利関係は明確に定めています

## 貢献の種類

どんな形の参加も歓迎します。

| 種類 | 方法 | 例 |
|:---|:---|:---|
| 感想を伝える | Issue | 「このシーンが特に刺さった」 |
| 誤字脱字を報告 | Issue または PR | 「○○という誤字があります」 |
| 文章の改善を提案 | Issue または PR | 「この比喩をこう変えると…」 |
| AI 制作プロセスについて質問 | Issue | 「このシーンはどんなプロンプトで？」 |

## Issue の出し方

1. [Issues](../../issues) ページを開く
2. "New issue" をクリック
3. 目的に合ったテンプレートを選ぶ
4. 内容を記入して送信

### ラベルの目安

| ラベル | 説明 |
|:---|:---|
| `感想` | 作品への感想・レビュー |
| `誤字脱字` | テキストの誤り |

| `質問` | AI 制作プロセスへの質問 |
| `技術` | サイト・CI の問題 |

## Pull Request の出し方

### 対象範囲

以下を対象とした PR を受け付けます：

- `src/` 内の原稿テキストの修正・改善
- `book.toml` や `.github/` 等の設定ファイルの改善
- `README.md` 等のドキュメント改善

### 手順

```sh
# 1. リポジトリを fork する

# 2. ローカルに clone する
git clone https://github.com/YOUR_USERNAME/confession-of-a-loser.git
cd confession-of-a-loser

# 3. ブランチを作成する
git checkout -b fix/typo-chapter1
# または
git checkout -b improve/prologue-expression

# 4. 変更を加える
# ... 原稿ファイルを編集 ...

# 5. mdbook でローカルプレビュー（任意）
mdbook serve

# 6. commit して push する
git add .
git commit -m "fix: 第一章の誤字を修正"
git push origin fix/typo-chapter1

# 7. GitHub 上で Pull Request を作成する
```

### ブランチ命名規則

| 接頭辞 | 用途 |
|:---|:---|
| `fix/` | 誤字脱字・バグ修正 |
| `improve/` | 文章の改善・表現の向上 |
| `docs/` | ドキュメントの修正 |
| `tech/` | 設定ファイル・CI の改善 |

### commit メッセージ

```
fix: 第一章「○○」の誤字を修正
improve: プロローグの比喩表現を改善
docs: README にローカル環境構築手順を追加
```

## コントリビューターライセンス同意（CLA）

**PR を提出する際は、以下に同意していただく必要があります。**

Pull Request テンプレートには確認用チェックボックスが含まれています。PR 送信前に必ずご確認ください。

### CLA の内容

PR を提出することにより、あなたは以下に同意します：

1. **著作権の譲渡**: 提出したコンテンツ（テキスト修正・追加・その他すべての変更）の著作権を **馬場祐平** に譲渡します
2. **商業利用の許可**: 著者は上記コンテンツを書籍出版・電子書籍・その他すべての形態での商業利用ができます
3. **オリジナリティの保証**: 提出するコンテンツは自分自身の創作であり、第三者のコンテンツをコピーしたものではありません
4. **クレジット**: 提出者の名前（GitHub ユーザー名）は、作品の「あとがき」または Contributors ページに記載されます

> **なぜ CLA が必要か**: 将来の書籍出版に際し、すべての寄稿部分の権利関係を明確にしておく必要があるためです。これにより、読者の貢献が書籍版にも反映される可能性が生まれます。

## レビュー権限について

| 役割 | 権限 |
|:---|:---|
| **著者（馬場祐平）** | 最終マージ権限・全 PR の承認・却下 |
| **共同創作者（K氏/しおめ）** | レビューコメント・承認（著者の確認後マージ） |
| **コントリビューター** | PR 提出・Issue 起票・コメント参加 |

レビューには時間がかかる場合があります。返答まで数日〜数週間かかることをご了承ください。

---

## クレジットについて

マージされた PR の提出者は、以下の方法でクレジットされます：

- GitHub の [Contributors ページ](../../graphs/contributors) への自動掲載
- 作品の「あとがき」または専用ページへの記載（書籍化の際を含む）

ありがとうございます。あなたの参加が、この作品をより良いものにします。
