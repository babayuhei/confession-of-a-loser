# 『失敗者の告白』オープンソース公開計画

## 1. プロジェクト概要

### 1.1 目的
1. **AI を活用した小説の作り方を共有する** — v10〜v23 に至る試行錯誤の過程そのものが、AI ナラティブ・エンジニアリングの実践記録である
2. **作品の改善に多くの人が関われる場を作る** — 読者からのフィードバック・プルリクエストを通じて、AI 協働創作の可能性を示す

### 1.2 公開形態
| 要素 | 方法 |
|:---|:---|
| ソースコード（原稿 Markdown） | GitHub パブリックリポジトリ |
| オンライン閲覧 | GitHub Pages + mdbook |
| フィードバック受付 | GitHub Issues |
| 改善提案受付 | GitHub Pull Requests |

### 1.3 権限構造
| 役割 | 人物 | 権限 |
|:---|:---|:---|
| **オーナー（Author）** | 馬場祐平 | 最終マージ権限・出版権・すべての意思決定権 |
| **共同創作者（Co-Creator）** | K氏（しおめ） | コードレビュー権限・マージ権限 |
| **コントリビューター** | 一般参加者 | Issue 起票・PR 提出・ディスカッション参加 |

## 2. ライセンス設計

### 2.1 二層ライセンス構造

作品の性質に応じて2つのライセンスを適用する。

| 対象 | ライセンス | 理由 |
|:---|:---|:---|
| **小説本文**（`src/` 内の全原稿） | CC BY-NC-ND 4.0 + CLA | 著作権保護と商業出版への道を確保しつつ、自由な閲覧を許可 |
| **プロセス文書・設定ファイル**（`book.toml`, workflows, テンプレート等） | MIT License | AI 小説制作の方法論は広く共有されるべき |

### 2.2 コントリビューターライセンス同意（CLA）

PRを提出する全コントリビューターは、以下に同意するものとする：

1. 提出したコンテンツの著作権を馬場祐平に譲渡する（著作権譲渡）
2. 馬場祐平は当該コンテンツを商業出版を含むあらゆる形態で使用できる
3. コントリビューターは PR 説明文内のチェックボックスで同意を表明する
4. コントリビューターのクレジットは「あとがき」または専用ページに記載する

> **設計意図**: オープンな貢献を歓迎しつつ、書籍化時の権利問題を未然に防ぐ

## 4. 運用フロー

### 4.1 Issue の種類

| ラベル | 用途 | 例 |
|:---|:---|:---|
| `感想` | 読後の感想やレビュー | 「第三章の佐伯のセリフが響いた」 |
| `誤字脱字` | テキストの誤り報告 | 「p.XX に漢字の誤りがある」 |
| `改善提案` | 内容の改善案 | 「比喩をこう変えると伝わりやすい」 |
| `質問` | AI 制作プロセスへの質問 | 「このシーンのプロンプト設計は？」 |
| `技術` | mdbook / CI / サイト関連 | 「モバイル表示が崩れる」 |

### 4.2 プルリクエストの流れ

```
1. コントリビューターが fork → ブランチ作成 → 原稿を修正
2. PR 提出時に CLA チェックボックスをオン（必須）
3. 馬場 or しおめ がレビュー
4. 承認 → main にマージ → GitHub Pages に自動デプロイ
```

### 4.3 ブランチ戦略

| ブランチ | 用途 |
|:---|:---|
| `main` | 公開版（GitHub Pages デプロイ元） |
| `draft/*` | 大幅改訂の作業ブランチ |
| `fix/*` | 誤字脱字等の軽微な修正 |

## 6. FAQ（想定質問）

**Q: この小説を自分のブログに転載してもいいですか？**
A: CC BY-NC-ND 4.0 に基づき、非営利・改変なし・クレジット表示の条件で共有可能です。

**Q: PRを出したら、その文章は私の権利ではなくなりますか？**
A: はい。CLA に基づき、マージされたコンテンツの著作権は馬場祐平に譲渡されます。ただし、コントリビューターとしてのクレジットは記載します。

**Q: AI でどうやって書いたのか詳しく知りたい**
A: Issue で質問していただければ、できる範囲でプロセスを共有します。将来的にプロセス解説ドキュメントの追加も検討しています。

**Q: 書籍版とオンライン版は同じですか？**
A: オンライン版は常に最新の状態を反映します。書籍版は特定時点のスナップショットとなります。
